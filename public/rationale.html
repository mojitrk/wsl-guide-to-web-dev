Rationale behind using WSL for web develepment, as opposed to X alternative.

Compared to a traditional Virtual Machine

WSL requires fewer resources (CPU, memory, and storage) than a full virtual machine. WSL also allows you to run Linux command-line tools and apps alongside your Windows apps and access to your Windows files from within Linux. You can use Windows apps and Linux command-line tools on the same set of files.

Isn't WSL a CLI-only thing?

Nope! Since the start of WSL, we could use X servers to run graphical apps from WSL itself. With WSL 2 comes WSLg, a built-in simplified X server that automatically adds your GUI apps to your host Windows Start Menu.
