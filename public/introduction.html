What is Windows Subsystem for Linux?

The Windows Subsystem for Linux lets developers run a GNU/Linux environment including most command-line tools, utilities, and applications directly on Windows, unmodified, without the overhead of a traditional virtual machine or dualboot setup.

What do you expect to accomplish at the end of this guide?


To install Windows Subsystem for Linux, set up a Linux development environment natively on Windows, access files from both file systems seamlessly, configure tools and software needed for web development and get an efficient workflow going in Windows itself.

What is WSL 2?

WSL 2 is a new version of the Windows Subsystem for Linux architecture that powers the Windows Subsystem for Linux to run ELF64 Linux binaries on Windows. Its primary goals are to increase file system performance, as well as adding full system call compatibility.

WHAT'S NEW IN

Windows Subsystem for Linux 2

Managed with Hyper-V

WSL 2 can be managed as a full fledged virtual machine using Hyper-V, while WSL 1 is merely a compatibility layer on top on Windows

Full Linux Kernel

WSL 2 uses a complete Linux kernel, albeit one modified by Microsoft.

File IO Performance

File IO performance has greatly increased in WSL 2, thanks to the OS files being saved in the WSL filesystem, instead of the Windows filesystem.

System Call Compatibility

WSL 2 offers full system call compatibility, allowing much needed apps like Docker to work on it.
