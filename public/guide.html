Don't be a clown,
use Visual Studio Code

Of course, you can use any code editor or IDE (vim and emacs users are quaking in their boots right now), but Visual Studio Code does have the best support for development in WSL. One with excellent remote development support is Jetbrains IDEs, with Gateway. Just make sure to link your interpreters, $PATHs and whatnot to the WSL's variant's path, not your Windows' one.

Install Visual Studio Code in Windows.
"$ winget install -e --id Microsoft.VisualStudioCode"

Install the "Remote Development - WSL" extension.
This will let you work in WSL with WSL-specific tooling.

Connect Visual Studio Code to WSL.
Click on the connection symbol on the bottom-left and select "WSL - Ubuntu".

Transfer any extensions you need to Remote WSL.
Since the Remote Development extension basically installs a extension server inside WSL, you'll have to transfer any extensions you need from Windows to WSL. Fortunately, it is just a one-click process, using the tiny cloud icon in the Extensions tab.

DONE

You're all set to go!
